# 企業登録機能の使い方

## 実装完了

案1（一般ユーザー向け企業登録）と案2（スーパー管理者による登録）の両方を実装しました。

## 1. 一般ユーザー向け企業登録

### アクセス方法
- URL: `/register`
- ログインページから「新規企業登録はこちら」リンクをクリック

### 登録内容
- **企業情報**
  - 企業名（必須）
  - 企業コード（必須、英数字のみ）
  - 企業メールアドレス（任意）
  - 電話番号（任意）
  - 住所（任意）

- **初期管理者アカウント**
  - 社員番号（必須）
  - 氏名（必須）
  - メールアドレス（必須）
  - パスワード（必須、8文字以上）

### 登録後の流れ
1. 企業情報と初期管理者アカウントが作成される
2. 企業設定がデフォルト値で作成される
3. ログインページにリダイレクト
4. 作成した管理者アカウントでログイン可能

## 2. スーパー管理者による企業登録

### アクセス方法
- URL: `/super-admin/companies`
- スーパー管理者としてログインが必要

### スーパー管理者の設定方法

現在、スーパー管理者は以下のいずれかの条件で判定されます：

1. `role` が `'super_admin'` の従業員
2. メールアドレスが `'superadmin@rakupochi.com'` の従業員

### スーパー管理者アカウントの作成方法

#### 方法1: データベースに直接作成

```sql
-- スーパー管理者アカウントを作成（例）
-- 注意: パスワードは適切にハッシュ化してください
INSERT INTO employees (
  "companyId",
  "employeeNumber",
  name,
  email,
  password,
  role,
  "isActive",
  "createdAt",
  "updatedAt"
)
VALUES (
  1, -- 既存の企業ID（適宜変更）
  'SUPER001',
  'スーパー管理者',
  'superadmin@rakupochi.com',
  '$2a$10$...', -- bcryptでハッシュ化したパスワード
  'super_admin',
  true,
  NOW(),
  NOW()
);
```

#### 方法2: シードデータで作成

`prisma/seed.ts` を修正してスーパー管理者を作成

### 機能

- **企業一覧表示**
  - 全企業の一覧
  - 従業員数、打刻数の表示
  - ステータス（有効/無効）の表示

- **企業登録**
  - 企業情報の入力
  - 初期管理者アカウントの作成
  - 企業設定の自動作成

- **企業編集**
  - 企業名、企業コード、連絡先の編集
  - ステータス（有効/無効）の変更

- **企業削除**
  - 企業と関連データの削除（Cascade）

## セキュリティ注意事項

1. **スーパー管理者アカウント**
   - 本番環境では適切な認証と権限管理を実装してください
   - メールアドレスベースの判定は開発用です

2. **企業コード**
   - 重複チェックが実装されています
   - 英数字のみ許可（大文字推奨）

3. **パスワード**
   - 8文字以上の要件
   - bcryptでハッシュ化して保存

## API エンドポイント

### 一般ユーザー向け
- `POST /api/companies/register` - 企業登録

### スーパー管理者向け
- `GET /api/super-admin/companies` - 企業一覧取得
- `POST /api/super-admin/companies` - 企業登録
- `PATCH /api/super-admin/companies/[id]` - 企業情報更新
- `DELETE /api/super-admin/companies/[id]` - 企業削除
