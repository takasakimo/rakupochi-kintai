# 自動通知生成（Cron Jobs）設定ガイド

## 概要

通知管理機能を自動化するため、Vercel Cron Jobsを使用して定期的にアラートを生成します。

## 実行スケジュール

現在の設定では、以下のスケジュールでアラートが自動生成されます：

- **毎日 9:00（JST）**: 打刻忘れアラートのチェック
- **毎日 18:00（JST）**: 残業時間アラート、連続勤務アラートのチェック

## 設定手順

### 1. 環境変数の設定

Vercelダッシュボードで以下の環境変数を設定してください：

1. Vercelダッシュボードにログイン
2. プロジェクトを選択
3. 「Settings」→「Environment Variables」を開く
4. 以下の環境変数を追加：

```
CRON_SECRET=your-random-secret-key-here
```

**CRON_SECRETの生成方法：**
```bash
openssl rand -base64 32
```

または、任意の長いランダムな文字列を使用してください。

### 2. Vercel Cron Jobsの確認

`vercel.json`に以下の設定が追加されています：

```json
{
  "crons": [
    {
      "path": "/api/cron/generate-alerts",
      "schedule": "0 9,18 * * *"
    }
  ]
}
```

この設定により、Vercelが自動的にcronジョブを設定します。

### 3. デプロイ後の確認

デプロイ後、Vercelダッシュボードで以下を確認できます：

1. 「Deployments」タブを開く
2. 「Functions」タブを開く
3. 「Cron Jobs」セクションで実行状況を確認

## 動作確認

### 手動実行（テスト用）

cronエンドポイントを手動で呼び出してテストできます：

```bash
curl -X GET https://your-domain.vercel.app/api/cron/generate-alerts \
  -H "Authorization: Bearer your-cron-secret"
```

### ログの確認

Vercelダッシュボードの「Functions」→「Logs」で実行ログを確認できます。

## 生成されるアラート

自動生成されるアラートの種類：

1. **残業時間アラート（40時間超）**
   - 今月の残業時間が40時間を超えた場合

2. **残業時間アラート（60時間超）**
   - 今月の残業時間が60時間を超えた場合（労働基準法の上限）

3. **打刻忘れアラート**
   - 平日（月〜金）に本日の出勤打刻が記録されていない場合
   - 9時のcron実行時にチェック

4. **連続勤務アラート**
   - 設定日数（デフォルト6日）以上連続で勤務している場合

## スケジュールの変更

スケジュールを変更する場合は、`vercel.json`の`schedule`を編集してください。

Cron式の例：
- `"0 9 * * *"` - 毎日9時
- `"0 9,18 * * *"` - 毎日9時と18時
- `"0 */6 * * *"` - 6時間ごと
- `"0 0 * * 1"` - 毎週月曜日の0時

## トラブルシューティング

### Cronが実行されない

1. 環境変数`CRON_SECRET`が正しく設定されているか確認
2. Vercelダッシュボードでcronジョブが有効になっているか確認
3. ログでエラーがないか確認

### 認証エラー

- `CRON_SECRET`が正しく設定されているか確認
- Vercel Cronからのリクエストには自動的に`Authorization`ヘッダーが付与されます

### アラートが生成されない

- 企業設定が正しく設定されているか確認
- 従業員の勤怠データが存在するか確認
- 既に未読のアラートが存在する場合は重複生成されません

## 注意事項

- Cron JobsはVercel Proプラン以上で利用可能です（無料プランでは利用できません）
- 実行時間は最大30秒です（`vercel.json`で設定）
- 全企業に対して処理を行うため、企業数が多い場合は実行時間が長くなる可能性があります
