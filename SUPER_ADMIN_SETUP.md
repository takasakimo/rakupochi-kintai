# スーパー管理者機能 セットアップガイド

## 概要

スーパー管理者機能により、複数の企業を管理し、各企業の従業員を登録できるようになりました。

## 機能

1. **スーパー管理者アカウントの作成**
2. **企業選択機能** - スーパー管理者がログインすると、管理する企業を選択できます
3. **企業の新規登録** - スーパー管理者が企業を新規登録できます
4. **従業員登録** - 選択した企業の従業員を登録できます

## セットアップ手順

### 1. スーパー管理者アカウントの作成

以下のコマンドを実行してスーパー管理者アカウントを作成します：

```bash
npm run create-super-admin
```

環境変数でカスタマイズ可能：

```bash
SUPER_ADMIN_EMAIL=superadmin@rakupochi.com
SUPER_ADMIN_PASSWORD=SuperAdmin123!
SUPER_ADMIN_NAME=スーパー管理者
```

デフォルト値：
- メールアドレス: `superadmin@rakupochi.com`
- パスワード: `SuperAdmin123!`
- 名前: `スーパー管理者`

### 2. ログイン

スーパー管理者アカウントでログインします。

### 3. 企業選択

ログイン後、企業選択画面が表示されます。管理する企業を選択してください。

### 4. 企業の新規登録

企業選択画面から「企業を新規登録」ボタンをクリックして、新しい企業を登録できます。

### 5. 従業員登録

企業を選択後、通常の管理者画面と同様に従業員を登録できます。

## 技術的な実装

### セッション管理

- スーパー管理者の場合は `companyId` が `null` になります
- 選択した企業IDは `selectedCompanyId` としてセッションに保存されます
- APIエンドポイントでは、スーパー管理者の場合は `selectedCompanyId` を使用します

### APIエンドポイント

以下のAPIエンドポイントがスーパー管理者に対応しています：

- `/api/admin/employees` - 従業員一覧・登録
- `/api/admin/employees/import` - CSVインポート
- `/api/admin/dashboard` - ダッシュボード
- `/api/super-admin/companies` - 企業管理
- `/api/super-admin/select-company` - 企業選択

### エラーハンドリング

スーパー管理者が企業を選択していない場合、以下のエラーが返されます：

```json
{
  "error": "企業が選択されていません"
}
```

この場合、企業選択画面にリダイレクトされます。

## 注意事項

1. **スーパー管理者のcompanyId**
   - スーパー管理者は「システム管理」という特別な企業（コード: `SYSTEM`）に所属します
   - 実際の管理対象企業は `selectedCompanyId` で指定されます

2. **セキュリティ**
   - スーパー管理者はすべての企業のデータにアクセスできます
   - 適切な権限管理を行ってください

3. **企業選択**
   - スーパー管理者がログイン後、必ず企業を選択する必要があります
   - 企業を選択しないと、管理画面にアクセスできません

## トラブルシューティング

### エラー: "企業が選択されていません"

**原因**: スーパー管理者が企業を選択していない

**解決方法**: 
1. `/super-admin/select-company` にアクセス
2. 管理する企業を選択

### エラー: "Forbidden"

**原因**: スーパー管理者権限がない

**解決方法**: 
1. スーパー管理者アカウントでログインしているか確認
2. `role` が `'super_admin'` になっているか確認

### スーパー管理者アカウントが作成できない

**原因**: 既にスーパー管理者が存在する、またはデータベース接続エラー

**解決方法**: 
1. 既存のスーパー管理者を確認
2. データベース接続を確認
3. 環境変数を確認
