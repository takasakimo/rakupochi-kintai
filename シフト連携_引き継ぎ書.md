### フェーズ2: 勤怠側 - 外部向けシフトAPI・認証 ✅

**目的**: リザーブが「APIキー」で勤怠のシフト一覧を取得できるようにする。

- [x] 2.1 APIキー（またはトークン）の保存先を決める（環境変数 1本 or 企業ごとDB）  
  → **実施**: 環境変数 `KINTAI_INTEGRATION_API_KEY` で1本管理（フェーズ3で企業ごと可に拡張可能）
- [x] 2.2 勤怠側に「連携用APIキー」を検証するミドルウェア or ヘルパーを実装  
  → **実施**: `lib/integration-auth.ts`（定数時間比較でタイミング攻撃対策）
- [x] 2.3 外部向けシフト取得エンドポイントを新設（例: `GET /api/integration/shifts`）  
  - クエリ: `start_date`, `end_date`, `company_code`（必須）。認証: `Authorization: Bearer <API_KEY>` または `X-API-Key`
  - レスポンス: `companyCode`, `companyId`, `companyName`, `shifts[]`（date, employeeEmail, employeeNumber, employeeName, startTime, endTime, breakMinutes, isOff, workLocation）
- [x] 2.4 既存の管理者用シフトAPI（`/api/admin/shifts`）のレスポンス形式を参考に、連携用の形式を定義・実装

- [ ] 2.5 レート制限 or 呼び出し上限の検討（任意）

**実装ファイル**:
- `lib/integration-auth.ts` - APIキー検証（Bearer / X-API-Key 対応）
- `app/api/integration/shifts/route.ts` - 外部向けシフト取得API
- `.env.example` - `KINTAI_INTEGRATION_API_KEY` の説明を追加

**デプロイ**: 2.3 まで実装した時点で **らくポチ勤怠** をデプロイし、APIキーで取得できることを確認する。
