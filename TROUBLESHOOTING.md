# ログインエラーの解決方法

## エラー: 「メールアドレスまたはパスワードが正しくありません」

このエラーが表示される主な原因は以下の通りです：

### 1. データベースにシードデータが投入されていない

データベースが作成されていても、テストアカウントが作成されていない可能性があります。

#### 解決方法

**ローカル環境で実行する場合：**
```bash
cd /Users/takasakimotonobu/rakupochi-kintai
npm run db:seed
```

**Vercel環境で実行する場合：**

Vercelでは直接シードコマンドを実行できないため、以下のいずれかの方法で対応：

#### 方法1: VercelのPostgresを使用する場合
1. Vercelダッシュボードで「Storage」タブを開く
2. データベースが作成されているか確認
3. 「Data」タブで直接データを挿入するか、以下のSQLを実行：

```sql
-- まず企業を作成
INSERT INTO companies (name, code, email, "isActive", "createdAt", "updatedAt")
VALUES ('テスト株式会社', 'TEST001', 'test@example.com', true, NOW(), NOW())
ON CONFLICT (code) DO NOTHING;

-- 企業IDを取得（上記で作成した企業のIDを使用）
-- パスワードは 'admin123' と 'employee123' をbcryptでハッシュ化したもの
-- 実際には、シードスクリプトを実行する必要があります
```

#### 方法2: 一時的な管理画面を作成
管理者用の従業員作成APIを作成して、手動でアカウントを作成する

#### 方法3: ローカルでシードを実行してからデータをエクスポート/インポート

### 2. データベース接続の問題

`DATABASE_URL`が正しく設定されていない、またはデータベースに接続できない可能性があります。

#### 確認方法
```bash
vercel env ls
```

`DATABASE_URL`が設定されているか確認してください。

### 3. パスワードの不一致

シードデータのパスワードと実際のパスワードが一致していない可能性があります。

#### 確認
シードファイル（`prisma/seed.ts`）で設定されているパスワード：
- 管理者: `admin123`
- 従業員: `employee123`

### 4. データベースマイグレーションが実行されていない

テーブルが作成されていない可能性があります。

#### 解決方法
```bash
# 環境変数を取得
vercel env pull .env.local

# マイグレーション実行
npx prisma migrate deploy
```

## 推奨される手順

1. **データベースの確認**
   - Vercelダッシュボードでデータベースが作成されているか確認
   - `DATABASE_URL`が正しく設定されているか確認

2. **マイグレーションの実行**
   ```bash
   vercel env pull .env.local
   npx prisma migrate deploy
   ```

3. **シードデータの投入**
   - ローカル環境で実行可能な場合は `npm run db:seed`
   - Vercel環境の場合は、上記の方法1または方法2を使用

4. **再デプロイ**
   ```bash
   vercel --prod
   ```

## 一時的な解決策

データベースがセットアップされていない場合、一時的に以下の方法でテストできます：

1. ローカル環境で開発サーバーを起動
2. ローカルでデータベースをセットアップ
3. シードデータを投入
4. ローカルで動作確認

その後、本番環境のデータベースをセットアップしてから、同じ手順を実行します。
